# Release Readiness

v0.90-v0.99 prepares Game Polish Lab for 1.0 Release Candidate readiness. v0.99 freezes scope for RC verification, but it does not publish the extension and does not complete pending manual release decisions.

Completed scope:

- v0.90: packaging metadata, package-check script, packaging notes, changelog
- v0.91: CI workflow for compile, tests, and package check
- v0.92: user-facing docs for the real workflow
- v0.93: user-facing command title polish and practical empty-state guidance
- v0.94: safety review docs and release-critical tests
- v0.95 Beta Dogfooding: report, checklist, and fixture-backed workflow tests across Idle Monster Farm, Sort Puzzle, and Cursor Arena
- v0.99 1.0 Release Candidate: frozen v1.0 scope, RC verification checklist, factual RC release notes, and release-candidate regression tests

Out of scope:

- publishing
- Marketplace publisher finalization
- license selection
- extension icon selection
- new adapters
- new visual surfaces
- gameplay changes in samples or fixtures
- live in-game overlay work
- v1.1+ feature expansion
- claiming pending manual decisions are completed

RC readiness still requires manual VS Code extension smoke testing from the Command Palette and manual decisions for publisher, license, icon, and Marketplace copy.
