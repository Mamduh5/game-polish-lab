# v0.7 Adapter Stabilization

v0.7 stabilizes the adapter series from the shared `VisualGameAdapter` contract through Sort Puzzle, Cursor Arena, Generic Phaser v2, theme transfer, screenshot annotation, and multi-game regression fixtures.

Game-specific adapters win over Generic Phaser when their stronger signals are present. Generic Phaser remains fallback-only and uses manual owner-file suggestions; it must not override Idle Monster Farm, Sort Puzzle, or Cursor Arena.

Direct apply remains narrow. Idle Monster Farm keeps its existing supported direct apply paths. Sort Puzzle, Cursor Arena, and Generic Phaser write generated configs only unless a runtime bridge is later implemented and tested. Source owner files are suspicious and require scoped visual-only fallback tasks.

Theme export/import is visual config portability. Export reads Game Polish Lab style configs, import writes generated style configs, incompatible surfaces are skipped or warned, and imported configs must not claim runtime application.

Screenshot annotation is manual. The user chooses a screenshot, rectangle, surface, target, and note. It does not run OCR, computer vision, AI visual analysis, or asset generation. Annotation outputs stay under `.game-polish-lab/**` and do not mark dashboard rows as applied.

Protected areas stay out of visual polish applies: save/state persistence, economy/balance/progression, level/rule/solver data, upgrade costs/effects, enemy HP/speed/spawn/damage, player/projectile/shooter/auto-shooter systems, ads/monetization, package churn, and unrelated adapter files.

v0.8 asset pipeline work has not started in v0.7. Do not treat asset dashboards, bounds normalization, style guide generation v2, manifest direct applies, or contact-sheet comparison as available v0.7 features.

## Manual Smoke Checklist

- Open Visual Tuning Dashboard in an Idle Monster Farm workspace.
- Open Visual Tuning Dashboard in a Sort Puzzle workspace.
- Open Visual Tuning Dashboard in a Cursor Arena workspace.
- Open Visual Tuning Dashboard in a Generic Phaser workspace.
- Verify adapter filter options.
- Verify rows and statuses do not overclaim applied/runtime integration.
- Verify Export Theme opens and writes theme files only.
- Verify Import Theme opens and writes generated configs only.
- Verify Annotate Screenshot opens and saves manual annotation/task files only.
- Verify fallback task generation is visual-only and scoped.
- Verify scope check blocks protected save/economy/progression/rule/ad/gameplay files.
