# v0.7 Migration Notes

v0.7 keeps the normal Game Polish Lab loop:

choose surface -> preview/tune -> save config/assets -> direct apply where supported -> record result.

The normal polish loop does not require Codex. Fallback tasks are only for unsupported games, one-time adapter creation or wiring, unusual integration, or structural work Game Polish Lab cannot safely apply itself.

## Supported Adapters

- `idle_monster_farm`
- `generic_phaser`
- `sort_puzzle`
- `cursor_arena`

## File Locations

- `.game-polish-lab/styles/**`
- `.game-polish-lab/themes/**`
- `.game-polish-lab/screenshot-notes/**`
- `.game-polish-lab/visual-recipes/**`
- `.game-polish-lab/field-notes/**`

## Guardrails

Direct apply is limited to validated generated style/config/asset paths. In short: direct apply is limited, guarded, and rollback-protected. Gameplay, save, economy, progression, rules, level data, solver logic, undo/hint logic, ad/monetization, Cursor Arena player/projectile systems, and Sort Puzzle rules/levels remain protected.

v0.8 asset pipeline work and v0.9 release packaging are not part of v0.7.
