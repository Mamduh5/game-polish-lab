# 1.0 Release Candidate

Date/version: 2026-06-29, Game Polish Lab 0.9.9 / v0.99 1.0 Release Candidate.

v0.99 is a release-candidate stabilization pass. It freezes the v1.0 scope around capabilities already implemented and stabilized. It does not publish the extension.

## Frozen v1.0 Scope

- Visual Tuning Dashboard.
- Slot/card, panel, button, reward toast, and background readability flows where existing.
- Asset import, validation, dashboard, contact-sheet, and manifest/config workflows where existing.
- Monster Farm adapter.
- Sort Puzzle adapter.
- Cursor Arena adapter.
- Generic Phaser adapter where existing.
- Direct apply where supported by existing safe config or manifest paths.
- Fallback task generation where direct apply is unsupported, structural, or unusual.
- Scope guard.
- Rollback snapshots and rollback history.
- Result tracking and field notes.
- Packaging, CI, docs, safety review, and beta dogfooding readiness.

## Deferred To v1.1+ Or Later

- Publishing to Marketplace.
- New adapters.
- New visual surfaces.
- Screenshot annotation as a production workflow.
- Live in-game dev overlay as a production workflow.
- Structural gameplay or layout editors.
- Automatic "make beautiful" AI design.
- Runtime integration for Sort Puzzle or Cursor Arena unless already supported by existing generated config paths.
- New asset pipeline feature families.
- Marketplace marketing/release work beyond factual RC readiness notes.

## Pending Manual Decisions

- Publisher identity.
- License selection.
- Extension icon.
- Marketplace copy.
- Manual VS Code smoke test from an Extension Development Host.

These decisions remain pending for RC. v0.99 does not mark them completed.

## Safety And Workflow

The intended v1.0 workflow remains:

1. Open the Visual Tuning Dashboard.
2. Choose a visual surface.
3. Preview and tune by eye.
4. Save config, asset metadata, or managed assets.
5. Direct apply only where an existing adapter reports support.
6. Run scope checks, use rollback, and record the result.

Codex remains fallback-only for unsupported adapters, one-time adapter creation, unusual integration work, or structural tasks. It is not the normal polish loop.

No gameplay, save, economy, progression, ad, rules, solver, level-data, player/projectile, shooter, monetization, or user-game logic is part of the frozen v1.0 scope.
